# 요구사항정의서

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | 코칭대화 롤플레이 챗봇 (1on1 Mate) |
| 목적 | AI와 함께 1on1 코칭대화를 연습하여 리더의 면담 스킬을 향상시키는 훈련 플랫폼 |

| 구분 | 기술 |

## 3. 시스템 구조

```
[사용자 (브라우저)]
       │
       ▼

       │
```

### FR-06. 대화 기록 DB 저장

| 항목 | 내용 |
|------|------|
| ID | FR-06 |
| 기능명 | 채팅 대화 영구 저장 |
| 우선순위 | 필수 |
| 상태 | ✅ 구현 완료 |
| 설명 | 롤플레이 채팅 메시지를 Supabase DB에 영구 저장 |

#### 데이터베이스 스키마

**conversations 테이블**

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | uuid (PK) | 자동 생성 (gen_random_uuid) |
| scenario_id | text | 시나리오 ID (goal / oneonone / performance) |
| created_at | timestamptz | 생성 시각 (자동) |

**messages 테이블**

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | uuid (PK) | 자동 생성 (gen_random_uuid) |
| conversation_id | uuid (FK) | conversations.id 참조 (CASCADE 삭제) |
| role | text | user / assistant |
| content | text | 메시지 내용 |
| created_at | timestamptz | 생성 시각 (자동) |

#### 저장 로직

1. 첫 메시지 전송 시 conversations 레코드 자동 생성
2. 이후 매 턴마다 user + assistant 메시지를 messages 테이블에 저장
3. conversationId를 클라이언트에 반환하여 동일 대화에 연속 저장

## 6. API 요구사항

### API-01. 채팅 API

| 항목 | 내용 |
|------|------|
| 경로 | POST /api/chat |
| 요청 파라미터 | messages (배열), systemPrompt (문자열), conversationId (uuid, 선택), scenarioId (문자열, 선택) |
| 응답 | { message: string, conversationId: string } |
| 외부 API | Google Gemini 2.5-flash REST API |
| DB 연동 | Supabase (conversations, messages 테이블) |

### API-02. 아바타 생성 API

| 항목 | 내용 |
|------|------|
| 경로 | POST /api/avatar |
| 요청 파라미터 | name, age, gender, jobTitle, personaType |
| 응답 | { imageUrl: string } |
| 외부 API | Google Gemini API (이미지 생성) |
